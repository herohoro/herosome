mdx からこんにちは！！！！<br/>
<br/>
このブログが使っているAwesomeさん。mdx版とnotion版とを選べまして、<br/>
ずっとハイブリッドにしたくって去年の今頃からウズウズしてたんです。<br/>
<br/>
<br/>
mosyaをやって、配列の感覚を強化させられたら、consoleでデータの動きが分かるようになりまして、ブログのいたるところにconsole爆弾を仕掛けてニヤニヤしていました。<br/>
<br/>
notion、apiなのであんまり動かすとエラーになっちゃうんで、mdx版にして開発環境で楽しんでおりました。<br/>
<br/>
mosya reactで型のヤバさを痛感し、いつも横にはプロパティの受け渡し場がありまして、その子はいずこから来た子なのかしら？と関数ずてに調べましたところ、それはそれは壮大なる物語が隠されていたのであります。<br/>
<br/>
関門には厳格な門番がいまして、お饅頭を差し入れれば怒りますし、少しラフな振る舞いをしましても見過ごしてはくれません。以前通った方にお伺いしながら必要なものを耳を揃えて収められまして、やっとの思いで通過いたしました。<br/>
<br/>
いろんな配列に助けられながら今があります。本当にありがとう。<br/>
<br/>
今日からmdxからでもnotionからでも記事が書けるようになりました。皆様のご協力あってのことです。<br/>
<br/>
mdxはコンポの兄貴も駆けつけてくれますし、御めかししたい気分の時はじっくりとcssを書くこともできます。<br/>
<br/>
もう少し季節が変わりましたら、md版も作ってobsidianともコラボ致そうかと根回し中です。<br/>
<br/>
<br/>
今回の改造で学べたこと：<br/>

<li> 関数のつながり</li>
<li> ファイルの属性により定義したsourceプロパティの活用</li>
<li> 必要な場面で必要な値を設置したりあだ名を付ける技</li>
<li> 原因究明のための遡りと蘇り</li>
<li> mdxの本文取得の沼からのfsへの変更もあり</li>
<li> 原因の大枠は自分の頭で考えることでchatGPTさんの助言が映える</li>
<li> どこまでできててどこからが問題なのか？の「きわ」を攻める</li>
<li> filePathまじ苦手....</li>
<br/>
<br/>
次やりたい：<br/>
- APIを極力使わないパフォーマンス上げ<br/>
- mx版のコンボ<br/>
<br/>
とりあえず、アイス買ってきます〜〜〜<br/>
